
## Проблема: мёртвые драконы пропадают из списка карт и из медпункта

### Жалоба игрока (S6AeK3R.tg)
1. Дракон умер → пропал из списка карт (нельзя воскресить).
2. Через время появился → отправил в медпункт → снова пропал.
3. Потом когда-нибудь появится после лечения.
4. Случай не единичный (зелёный, огненный драконы).

### Гипотезы (нужно проверить в коде)

**Гипотеза A: Мёртвые драконы фильтруются из списка карт.**
Memory `dead-hero-visibility-and-removal` говорит: «Dead heroes (currentHealth ≤ 0) must remain visible in team selection but become unselectable». Это про героев в DeckSelection. Но в **MedicalBay** и в общем списке карт мёртвые драконы могут отфильтровываться где-то на пути `useCards` → фильтры по `current_health > 0`.

**Гипотеза B: Карта в медпункте удаляется/скрывается из основного списка.**
Когда карту отправляют на лечение, она попадает в `medical_bay` и, возможно, исчезает из основного списка `useCards`/`useCardInstances` (фильтр «не показывать карты, находящиеся в медпункте»). Это объясняет «отправил → пропал».

**Гипотеза C: Расхождение `card_type` для драконов.**
Драконы хранятся как `card_type='dragon'`, а UI ожидает `'pet'` (или наоборот). Если фильтр медпункта/списка карт ищет только `'hero'`, драконы могут отваливаться. Memory `card-type-normalization-utility` подтверждает наличие проблем с маппингом типов.

**Гипотеза D: Realtime/refetch race condition.**
После смерти дракона/отправки в медпункт срабатывает оптимистичный апдейт + realtime, и карта временно исчезает из кэша React Query, пока не придёт свежий рефетч. Memory `instance-refresh-policy` и `dungeon-claim-key-protection` намекают на похожие проблемы.

### Что планирую исправить (после диагностики)

В зависимости от того, что найдём:
- **Если фильтр в MedicalBay скрывает драконов** → расширить фильтр, чтобы включал `card_type IN ('dragon','pet','hero','character')`.
- **Если карта пропадает после отправки в медпункт** → показывать карты, находящиеся в медпункте, в специальной секции (с пометкой «лечится»), а не убирать совсем.
- **Если мёртвые драконы фильтруются из списка карт** → применить тот же принцип, что и для героев (видны, но неактивны), плюс гарантировать, что они доступны для воскрешения.
- **Если race condition** → добавить блокировку рефетча на 1–2 сек после оптимистичного апдейта (по аналогии с `claimInProgressRef`).
